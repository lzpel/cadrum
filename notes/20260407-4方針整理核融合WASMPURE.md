● CadQueryと比較して、広く使ってもらうには不自然・改善すべき点を以下にまとめます。

  ---
  1. 命名の不自然さ

  box_from_corners → box_ or box_by_corners

  CadQueryでは Workplane("XY").box(10, 20, 30)
  のようにシンプル。from_cornersは2点指定という実装詳細が名前に出すぎ。多くのユーザーは「幅・高さ・奥行き」で箱を作りたい。

  color_paint → with_color or color

  「paint」は動詞としてCAD文脈で馴染みがない。CadQueryやOpenSCADでも color("red") のような命名。Some(Color::from_str("#4a90d9")?)
  という呼び出しも冗長。

  half_space

  一般ユーザーが直接使うプリミティブとしては抽象的すぎる。ドキュメントなしでは何を作るか分からない。

  approximation_segments / approximation_segments_ex

  Edgeのテッセレーション用だが、名前が長く意図が不明瞭。tessellate や to_points の方が直感的。_ex サフィックスもC言語的で Rust らしくない。     

  tshape_id

  OCCT内部の TopoDS_TShape ポインタを露出している。公開APIとしては不適切。内部IDが必要なら id() で十分。

  ---
  2. 使用方法の不自然さ

  Boolean操作がメソッドチェーンできない

  // 現状: 静的メソッド + スライス + 変換チェーン
  Boolean::union(&[box_], &[cyl])?.into_solids()

  // CadQuery風に期待される形:
  box_.union(&cyl)
  // または
  box_.fuse(&cyl)
  最大の問題点。CADライブラリで最も頻繁に使う操作なのに、最も冗長。

  into_solids() の強制

  Boolean結果を使うたびに .into_solids() が必要。多くの場合、結果は1つのSolidなのに Vec<Solid>
  を返す。CadQueryのように単一Solidを返し、複数の場合だけ solids() で取得する方が自然。

  translateがselfを消費、scaledは借用 — 一貫性がない

  solid.translate(v)   // self を消費
  solid.scaled(c, f)   // &self を借用して新しいSolidを返す
  命名も translate vs scaled（過去分詞）で不統一。どちらかに統一すべき：
  - 全部消費型: translate, rotate, scale, mirror
  - 全部借用型: translated, rotated, scaled, mirrored

  rotate の引数が多すぎる

  solid.rotate(axis_origin: DVec3, axis_direction: DVec3, angle: f64)
  3つの引数のうち2つがDVec3で、どちらが原点でどちらが方向か読みにくい。CadQueryでは rotate((0,0,0), (0,0,1), 45) だが、メソッド名で軸を示す     
  rotate_x(angle) 等のショートカットがある。

  I/Oがフリー関数のみ

  // 現状
  io::write_step(&solids, &mut file)?;

  // より自然
  solids.write_step("output.step")?;
  // または
  solid.to_step("output.step")?;
  Read/Write traitを使ってファイルパスを直接受け取るヘルパーがあると使いやすい。

  Faceからの押し出しがFaceのメソッド

  Face::from_polygon(&points)?.extrude(DVec3::Z * 10.0)?
  これ自体は悪くないが、CadQueryでは Workplane → sketch → extrude というフローが確立されている。2Dスケッチ→3D変換のワークフローが薄い。

  ---
  3. 欠けている便利API（広く使ってもらうために）

  ┌───────────────────────────────────────────────────────┬──────────────────────────────────────┐
  │                         不足                          │           CadQueryでの対応           │
  ├───────────────────────────────────────────────────────┼──────────────────────────────────────┤
  │ Solid::box_(w, h, d) のようなサイズ指定コンストラクタ │ box(length, width, height)           │
  ├───────────────────────────────────────────────────────┼──────────────────────────────────────┤
  │ solid.fuse(other) / solid.cut(other)                  │ メソッドチェーンでBoolean            │
  ├───────────────────────────────────────────────────────┼──────────────────────────────────────┤
  │ solid.fillet(radius) / solid.chamfer(dist)            │ エッジ処理の基本操作                 │
  ├───────────────────────────────────────────────────────┼──────────────────────────────────────┤
  │ rotate_x, rotate_y, rotate_z                          │ 軸回転ショートカット                 │
  ├───────────────────────────────────────────────────────┼──────────────────────────────────────┤
  │ Workplane / 2Dスケッチ機能                            │ Workplane("XY").circle(r).extrude(h) │
  └───────────────────────────────────────────────────────┴──────────────────────────────────────┘

  ---
  4. 優先度の高い改善提案

  1. Booleanをメソッドチェーンに: solid.union(&other) — これだけでユーザー体験が劇的に改善
  2. transform系の一貫性: 全て self 消費 + 動詞原形（translate, rotate, scale, mirror）に統一
  3. color_paint → color or with_color: Some() なしで solid.color("red") と書けるように
  4. box_from_corners に加え Solid::box_(w, h, d) を追加: 最も使うプリミティブなので簡潔に
  5. approximation_segments_ex → ビルダーパターンか引数名付き構造体に: _ex パターンは避ける

pub trait SolidTrait: Sized + Clone {
    // 単体だけ
	// --- Constructors --- 
	fn cube(x: f64, y: f64, z: f64) -> Self;
	fn sphere(radius: f64) -> Self;
	fn cylinder(r: f64, axis: DVec3, h: f64) -> Self;
	fn cone(r1: f64, r2: f64, axis: DVec3, h: f64) -> Self;
	fn torus(r1: f64, r2: f64, axis: DVec3) -> Self;
	fn half_space(plane_origin: DVec3, plane_normal: DVec3) -> Self;

    // 複数で出来たら嬉しい
	// --- Transforms ---
	fn translate(self, translation: DVec3) -> Self;
	fn rotate(self, axis_origin: DVec3, axis_direction: DVec3, angle: f64) -> Self;
	fn rotate_x(self, angle: f64) -> Self { self.rotate(DVec3::ZERO, DVec3::X, angle) }
	fn rotate_y(self, angle: f64) -> Self { self.rotate(DVec3::ZERO, DVec3::Y, angle) }
	fn rotate_z(self, angle: f64) -> Self { self.rotate(DVec3::ZERO, DVec3::Z, angle) }
	fn scale(self, center: DVec3, factor: f64) -> Self;
	fn mirror(self, plane_origin: DVec3, plane_normal: DVec3) -> Self;
	fn clean(&self) -> Result<Self, Error>;

    // 複数で出来たら嬉しい
	// --- Queries ---
	fn volume(&self) -> f64;
	fn bounding_box(&self) -> [DVec3; 2];
	fn contains(&self, point: DVec3) -> bool;
	fn shell_count(&self) -> u32;

    // 不要、そもそもこの関数自体が必要か怪しい
	// --- Topology ---
	fn faces(&self) -> Vec<Face>;
	fn edges(&self) -> Vec<Edge>;

    // 必要、ただし名前は変えたい
	// --- Color ---
	#[cfg(feature = "color")]
	fn color_paint(self, color: Option<Color>) -> Self;
	#[cfg(feature = "color")]
	fn color(&self) -> Option<Color>;
}

//非公開 struct Solidはこれを実装しているべき

trait SolidStruct: SolidExt{
	// --- Constructors --- 
	fn cube(x: f64, y: f64, z: f64) -> Self;
	fn sphere(radius: f64) -> Self;
	fn cylinder(r: f64, axis: DVec3, h: f64) -> Self;
	fn cone(r1: f64, r2: f64, axis: DVec3, h: f64) -> Self;
	fn torus(r1: f64, r2: f64, axis: DVec3) -> Self;
	fn half_space(plane_origin: DVec3, plane_normal: DVec3) -> Self;
	// --- Topology ---
	fn faces(&self) -> Vec<Face>;
	fn edges(&self) -> Vec<Edge>;
}
// 公開 配列であれ単体であれ共通して利用できるべき関数
// impl SolidExt for Solidはsolid.rsで実装する
// impl <T: SolidStruct> SolidExt for Vec<T> {} // Vec<Solid>でも使えるように
// impl <T: SolidStruct> SolidExt for [T] {} // 配列でも使えるように
pub trait SolidExt{
    // 複数で出来たら嬉しい
	// --- Transforms ---
	fn translate(self, translation: DVec3) -> Self;
	fn rotate(self, axis_origin: DVec3, axis_direction: DVec3, angle: f64) -> Self;
	fn rotate_x(self, angle: f64) -> Self { self.rotate(DVec3::ZERO, DVec3::X, angle) }
	fn rotate_y(self, angle: f64) -> Self { self.rotate(DVec3::ZERO, DVec3::Y, angle) }
	fn rotate_z(self, angle: f64) -> Self { self.rotate(DVec3::ZERO, DVec3::Z, angle) }
	fn scale(self, center: DVec3, factor: f64) -> Self;
	fn mirror(self, plane_origin: DVec3, plane_normal: DVec3) -> Self;
	fn clean(&self) -> Result<Self, Error>;

    // 複数で出来たら嬉しい
	// --- Queries ---
	fn volume(&self) -> f64;
	fn bounding_box(&self) -> [DVec3; 2];
	fn contains(&self, point: DVec3) -> bool;
	fn shell_count(&self) -> u32;
    // 複数で出来たら嬉しい、ただし名前は変えたい
	// --- Color ---
	#[cfg(feature = "color")]
	fn color_paint(self, color: Option<Color>) -> Self;
	#[cfg(feature = "color")]
	fn color(&self) -> Option<Color>;
    // --- Boolean --- メソッドチェーンで書けるように簡略版も用意
	fn union_with_metadata(self, tool: impl IntoIterator<Item = &T>) -> Result<(Vec<T>, [Vec<u64>;2]), Error>;
	fn subtract_with_metadata(self, tool: impl IntoIterator<Item = &T>) -> Result<(Vec<T>, [Vec<u64>;2]), Error>;
	fn intersect_with_metadata(self, tool: impl IntoIterator<Item = &T>) -> Result<(Vec<T>, [Vec<u64>;2]), Error>;
    fn union(self, tool: impl IntoIterator<Item = &T>) -> Result<Vec<T>, Error> {self.union_with_metadata(tool)?.0}
    fn subtract(self, tool: impl IntoIterator<Item = &T>) -> Result<Vec<T>, Error> {self.subtract_with_metadata(tool)?.0}
    fn intersect(self, tool: impl IntoIterator<Item = &T>) -> Result<Vec<T>, Error> {self.intersect_with_metadata(tool)?.0}
}
