## 追記

### operations のエンコーディングを DNF (DIMACS形式) に差し替え

上記の `-1=union, -2=intersect, -3=subtract` を operator code として混ぜる方式は parser が必要で読みづらい。代わりに **任意のブール式を DNF (Disjunctive Normal Form / 積和形) に正規化** してから `Vec<i64>` に flat 化する。

#### なぜ DNF か

`BOPAlgo_CellsBuilder::AddToResult(toTake, toAvoid)` の意味は

```
result = ⋃ᵢ ( (⋂ s∈takeᵢ s) ∖ (⋃ s∈avoidᵢ s) )
```

これは DNF そのもの。AddToResult 1回 = DNF の1積項に1対1で対応するので、Rust 側で DNF を作っておけば C++ 側のロジックは「clause を読んで AddToResult を呼ぶ」だけになる。

#### エンコーディング (DIMACS CNF と同じ規約)

- `+i` (i≥1): `solids[i-1]` を take
- `-i` (i≥1): `solids[i-1]` を avoid
- `0`: clause 終端
- 末尾の `0` は必須

例 (solids = [A, B, C, D]):

| 式 | clauses |
|---|---|
| `A + B + C` | `[1, 0, 2, 0, 3, 0]` |
| `A + B - C` | `[1, -3, 0, 2, -3, 0]` |
| `(A + B) * C` | `[1, 3, 0, 2, 3, 0]` |
| `A*B + C*D` | `[1, 2, 0, 3, 4, 0]` |
| `A - B*C` | `[1, -2, 0, 1, -3, 0]` |

#### 型

```rust
pub struct Boolean<S: SolidStruct> {
    solids:  Vec<S>,
    clauses: Vec<i64>,   // DIMACS-flat DNF, 0-terminated per clause
}
```

`isize` ではなく `i64` 固定 (32/64bit target 間の FFI 事故を避けるため、かつ `cxx::CxxVector<i64>` にそのまま載るため)。`Clause` 構造体は作らない (AGENTS.md「構造体を増やす方向より減らす方向を優先」)。

#### 演算子の合成規則 (全部 DNF 上で閉じる)

`Boolean op Boolean → Boolean`:

- `a + b` (union): clause を連結
- `a * b` (intersect): clause の直積 (各組み合わせで take/avoid をマージ)
- `a - b` (subtract = `a ∩ ¬b`): `¬b` の DNF 化 (各 clause で take↔avoid 入れ替え、その後直積) してから `a` と intersect

`-` は展開が膨れるが、CellsBuilder は同一 solid の重複参照に強い (交差計算は1回しか走らない) ので性能上の損はない。

#### Rust 側のイテレーション

```rust
for clause in self.clauses.split(|&lit| lit == 0).filter(|c| !c.is_empty()) {
    // clause: &[i64]
}
```

`slice::split` 一発。

#### C++ 側 (FFI 境界)

```cpp
void boolean_build(
    const std::vector<TopoDS_Shape>& solids,
    const std::vector<int64_t>&     clauses,
    TopoDS_Shape&                   out)
{
    BOPAlgo_CellsBuilder cb;
    TopTools_ListOfShape args;
    for (auto& s : solids) args.Append(s);
    cb.SetArguments(args);
    cb.Perform();

    TopTools_ListOfShape take, avoid;
    const int material = 1;
    for (int64_t lit : clauses) {
        if (lit == 0) {
            if (!take.IsEmpty() || !avoid.IsEmpty()) {
                cb.AddToResult(take, avoid, material);
                take.Clear(); avoid.Clear();
            }
        } else if (lit > 0) take.Append(solids[ lit - 1]);
        else                avoid.Append(solids[-lit - 1]);
    }
    cb.RemoveInternalBoundaries();
    out = cb.Shape();
}
```

FFI 境界を渡るのは `CxxVector<TopoDS_Shape>` と `CxxVector<i64>` の2本だけ。入れ子も struct もないので cxx bridge が最小。

#### SolidStruct trait の整理

既存の

```rust
fn boolean_union<'a, 'b>(a: ..., b: ...) -> Result<Vec<Self>, Error> where ...;
fn boolean_subtract<'a, 'b>(a: ..., b: ...) -> Result<Vec<Self>, Error> where ...;
fn boolean_intersect<'a, 'b>(a: ..., b: ...) -> Result<Vec<Self>, Error> where ...;
```

は削除し、唯一の FFI 通路として

```rust
fn boolean_build(solids: &[Self], clauses: &[i64]) -> Result<Vec<Self>, Error>;
```

に統合。`Solid::union/subtract/intersect` の公開ラッパーが必要なら、`Boolean::from(&a) + &b` 等の薄い糖衣として外側に置く。

#### 注意点

- `+0`/`-0` の曖昧性: 1-indexed なので `0` は禁則値。`debug_assert!(lit != 0)` を構築時に。
- `solids` の所有: `Boolean` が `Vec<S>` を所有する。`+`/`-`/`*` 演算子内で clone するので参照寿命の問題は発生しない。`TopoDS_Shape` の clone は handle の浅いコピーで安価。
- 空 clause (`[..., 0, 0, ...]`): 数学的には恒偽 (empty intersection = universe ではなく empty set として扱う)。C++ 側で if で弾く。
- 重複リテラル (`[1, 1]` や `[1, -1]`): 正規化せず CellsBuilder に渡しても動く (前者は冗長、後者は空集合)。性能を気にするなら build 時に簡約。

#### まとめ

- 中間表現 = DIMACS-flat DNF = `Vec<i64>` + 0終端
- 業界標準形式なので俺ルールにならない
- `Clause` 構造体不要、FFI に struct を渡す必要なし
- cxx ブリッジは `CxxVector<i64>` 1本で済む
- 演算子合成は DNF 上で閉じる
