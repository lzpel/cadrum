# cxx置換対応表

cxx/cxx-build クレートを排除した際に、cxx が提供していた各機能を何で置き換えたかの対応表 (#CHANGELOG Unreleased 参照)。

動機: cxx クレートは自身の build.rs で `src/cxx.cc` を無条件コンパイルするため、wrapper をプリビルト配布しても消費者に wasm C++ ツールチェインが要求され続ける。bindgen も調査したが（build.rs は C を含まない）、build-dependency にすると全消費者に libclang が必須になるため採用せず、**bindgen 出力相当の手書き ffi.rs をコミットする方式**にした。

## 対応表

- **`#[cxx::bridge]` proc-macro（Rust 側宣言の生成）**
  → `src/occt/ffi.rs` にコミットした手書きの `extern "C"` raw 宣言（`mod raw`）＋旧ブリッジと同名・同シグネチャの安全ラッパ関数。呼び出し側の変更は型名と `pin_mut()` 除去のみ。

- **cxx-build によるブリッジ C++ コード生成 (`ffi.rs.h` / `ffi.rs.cc`) とコンパイル**
  → `cpp/wrapper.h` を純 C ヘッダ化（opaque 型 + `cadrum_` プレフィクス関数）し、`cpp/wrapper.cpp` 末尾に extern "C" シム層を追加。コンパイルは素の `cc::Build`（cc は既存の build-dependency）。

- **`cxx::UniquePtr<T>`（C++ ヒープ所有権の Rust 側管理）**
  → 自前の所有型 `ffi::Ptr<T>`。Drop で `cadrum_shape_delete` 等の delete 関数を呼ぶ。null 許容 (`is_null`)、Deref は null でパニック（UniquePtr と同挙動）、`Send` のみ実装で `!Sync`（OCCT Handle の参照カウントが非アトミックなため）という旧特性を維持。

- **opaque 型 (`type TopoDS_Shape;` 等)**
  → `#[repr(C)] struct TopoDS_Shape { _opaque: [u8; 0] }` の bindgen 流 opaque 構造体。

- **`CxxVector<T>`（std::vector の Rust 側ビュー: iter / is_empty / pin_mut().push）**
  → opaque 型 `ShapeVec` / `FaceVec` / `EdgeVec` ＋ C ABI の `*_vec_new` / `*_vec_push` / `*_vec_len` / `*_vec_get` / `*_vec_delete`。Rust 側は `iter()` / `is_empty()` を `len`/`get` 越しに実装。

- **`rust::Slice<const T>`（Rust スライスの C++ 側受け渡し）**
  → C ABI では `(const T* ptr, size_t len)` の 2 引数に展開。C++ 内部実装向けには wrapper.cpp 内の自前 `Slice<T>` テンプレート（`size()` / `operator[]` / `data()`）で既存関数本体を無改変のまま維持。

- **`rust::Vec<T>`（C++ から Rust `Vec` への出力: out_history, out_ids, out_rgb, 戻り値 Vec<f64>）**
  → C++ 側は `std::vector<T>` に組み立て、シムが Rust 定義の extern "C" コールバック `cadrum_{u32,u64,f32,f64}_extend(vec, ptr, len)` で `*mut Vec<T>`（`void*` 経由）へ一括コピー。

- **共有構造体 `MeshData`（Vec フィールド持ち struct の境界越え）**
  → C ABI は `cadrum_mesh_shape(..., out_vertices, out_normals, out_indices, out_face_ids) -> bool` の 4 out-vec に分解。Rust の安全ラッパが従来どおりの `MeshData` 構造体（純 Rust 定義）へ再組み立て。

- **`extern "Rust"`（RustReader/RustWriter と rust_reader_read/rust_writer_write の C++ 公開）**
  → `#[no_mangle] extern "C" fn cadrum_reader_read(void*, *mut u8, usize) -> usize` / `cadrum_writer_write`（src/occt/stream.rs）。C++ の streambuf は `void*` の opaque ポインタを保持して直接呼ぶ。

- **`Pin<&mut T>`（可変参照の安全な受け渡し）**
  → 通常の `&mut T`（安全ラッパ内で `*mut T` に変換）。`pin_mut()` 呼び出しは全廃。

- **名前空間分離 (`namespace = "cadrum"` による C++ シンボル管理)**
  → extern "C" 関数名への `cadrum_` プレフィクスで衝突回避（C ABI に名前空間はないため）。

- **link-cplusplus（C++ 標準ライブラリのリンク行の発行）**
  → cc クレートの既定挙動（target 別に `stdc++`/`c++`、`CXXSTDLIB` env で上書き可）。wasm cross イメージは既に `ENV CXXSTDLIB=c++` を設定済みで、link-cplusplus が出していた `-l c++` と同一挙動になることを cc ソースで確認。

- **例外の境界処理（cxx は非 Result 関数で例外→terminate）**
  → wrapper.cpp の各実装が従来どおり `Standard_Failure` を catch して nullptr/false を返す。extern "C" 境界を例外が越えない構造は不変。

## 置き換えで消えたもの・増えたもの

- 消: `cxx`（依存）、`cxx-build`（build 依存）、cxx の `src/cxx.cc` 無条件コンパイル、生成ヘッダ `ffi.rs.h`。
- 増: 依存クレートはゼロ増。
- 注意（同期義務の性質変化）: wrapper.h ↔ ffi.rs の手動同期義務は**旧構成でも存在した**（どちらも手書きで自動書き換えは無い）。ただし旧構成では cxx-build 生成の `ffi.rs.h` を wrapper.cpp が include するため、シグネチャ不一致は C++ コンパイルエラーとして**機械検証**されていた。新構成の extern "C" はリンカがシンボル名しか照合しないため、不一致はコンパイルエラーにならず**実行時 UB**になる。失われたのは自動化ではなく不一致検出。緩和策: ヘッダは bindgen で解析可能な純 C 形を維持しており、`bindgen cpp/wrapper.h` の出力と ffi.rs の raw 宣言を突き合わせる検査を CI に足せる。
